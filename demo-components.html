Component System Demo

This page demonstrates the new component system for consistent layout across all PhoenixBird Capital pages.

How It Works

The navbar and footer are now stored as separate HTML components:

  • components/navbar.html - Contains the navigation bar markup
  • components/footer.html - Contains the footer markup

These components are loaded dynamically using JavaScript:

  • js/components.js - Handles loading and displaying components

Implementation Steps

To update existing pages to use this component system:

  1. Replace navbar and footer sections with container divs
  2. Add the components.js script before closing body tag

Example HTML Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <!-- Meta tags, title, CSS links -->
</head>
<body>
    <header>
        <!-- Navbar will be loaded here -->
        <div id="navbar-container"></div>
    </header>
    
    <main>
        <!-- Page content here -->
    </main>
    
    <!-- Footer will be loaded here -->
    <div id="footer-container"></div>
    
    <!-- Scripts -->
    <script src="js/api.js"></script>
    <script src="js/navigation.js"></script>
    <script src="js/components.js"></script>
    <script src="js/main.js"></script>
</body>
</html>

Benefits

  • Consistent navigation and footer across all pages
  • Easier maintenance - update one file instead of every page
  • Reduced code duplication
  • Automatic highlighting of active navigation items
  • Dynamic content based on authentication status
